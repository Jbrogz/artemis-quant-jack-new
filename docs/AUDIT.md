## 3. Survivorship: terminal collapse cohorts

Assets with a terminal >90% drawdown (last observed price ≤10% of all-time peak):

| Cohort | Count | Definition |
|---|---|---|
| **Total terminal collapses** | **238** | Last price ≤10% of all-time peak |
| Delisted cohort | 15 | `delisted_asof` ever fired — price reporting stopped |
| Zombie cohort | 223 | Still printing at near-zero (never formally delisted) |

**All 238 collapsing assets are kept in the panel.** Their crash returns are
carried as realized simple returns in `data/returns/holding_returns.parquet`;
no loss is silently dropped. The `lunc` (LUNA) asset is included with its
~−99.99% terminal return.

**Delisted cohort (15 assets):** price reporting lapsed more than
`LISTING_STALENESS_DAYS` (7 days) past the last observed price. The
`delisted_asof` flag fires point-in-time on each date the coin is silent,
so the returns layer books the crash return on the actual last-print date, not
a fabricated date.

**Zombie cohort (223 assets):** still reporting prices today but permanently
near zero. Their `delisted_asof` flag never fires (they are technically still
alive), yet they exhibit a >90% realized drawdown from peak. They remain in
the eligible-candidate pool until the staleness filter eventually removes them,
which is the correct point-in-time behavior.

An asset is left-censored if its first observed price equals the panel start
date (2018-01-01). This means the asset may have listed before the pull window
began — its true listing date is unknown. Left-censored assets are included
in the panel (dropping them would be survivorship bias), but their pre-2018
history is unobservable. The `left_censored` flag is carried in the panel for
downstream disclosure.

## 5. Residual survivorship — unestimable lower bound

**The Artemis `/asset` catalog is an as-of-today snapshot with no
listing/delisting dates.** Assets that were fully purged from the Artemis
platform before this query (i.e. coins that collapsed and were subsequently
removed from the catalog entirely) are **unrecoverable**. Their price series
no longer exist anywhere in the Artemis API, so they cannot be included even
with the crash-carry approach.

This is the irreducible residual survivorship bias of the dataset. It cannot
be quantified from within the Artemis API alone. Per spec §3.6/§10, we:

1. Include every catalog asset that shows any price history, including
   collapsed coins, regardless of current eligibility.
2. Carry the realized terminal crash return for every collapsed coin.
3. Disclose explicitly that purged-dead-coins represent an unestimable
   lower bound on survivorship bias — the number of purged names is unknown.

The reported factor returns should therefore be interpreted as **upper bounds**
relative to a true point-in-time universe that could, hypothetically, recover
all coins ever listed on Artemis.
